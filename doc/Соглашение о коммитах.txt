Комментарий к коментарию должен начинаться с одного из заданных префиксов :
[CS] - изменения code style не влияющие на конечного пользователя.
[UI] - изменения GUI не изменяющие функционала приложения.
[PROJECT] - изменения в структуре проекта, внутреннем расположении файлов проекта, добавление\изменение\создание служебных файлов, не попадающих в конечный продукт
[FEATURE] - изменения, добавляющие новый функционал или меняющие существующий, или удаление существующего функционала
[BUGFIX] - исправление ошибок существующем функционале

Предпочтительный язык комментариев - английский.
Рекомендуется в рамках одного коммита делать изменения одного из представленных выше типов.
Если коммит затрагивает несколько типов тогда необходимо вначале описать все типы изменений через пробел ([UI] [CS] [FEATURE]), затем поставить ":", 
и потом через "; " описать изменения по каждому из типов правок.

Пример 1:
[FEATURE] [CS] : change and add new field in statistic; change widget names

Пример 2:
[FEATURE] add eventView
