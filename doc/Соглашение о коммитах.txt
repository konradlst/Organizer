Комментарий к коментарию должен начинаться с одного из заданных префиксов :
[CS] - изменения code style не влияющие на конечного пользователя.
[UI] - изменения GUI не изменяющие функционала приложения.
[PROJECT] - изменения в структуре проекта, внутреннем расположении файлов проекта, добавление\изменение\создание служебный файлов не попадающих в конечный продукт
[FEATURE] - изменения добавляющие новый функционал или меняющий существующий или удаление существующего функционала

Предпочтительный язык комментариев - английский.
Рекомендуется в рамках одного коммита делать изменения одного из представленных выше типов.
Если коммит затрагивает несколько типов тогда необходимо первой строкой, оканчивающейся ":", описать все типы изменений через пробел ([UI] [CS] [FFATURE]), 
и в последующих строках, разделенных ";" и переносом строки, описать изменения по каждому из типов правок.

Пример 1:
[FEATURE] [CS] :
change and add new field in statistic;
change widget names

Пример 2:
[FEATURE] add eventView
