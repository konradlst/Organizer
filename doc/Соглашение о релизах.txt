Версия приложения состоит из 4-х цифр (+ тип релиза), где :
1 - мажорная версия, изменяется при координальных изменениях в приложении.
2 - минорная версия, изменяется в введение нового функционала, меняется в каждом релизе.
3 - стабилизирующая версия, изменяется(добавляется) при выпуске пакета багфиксов к текущей версии.
4 - версия сборки (в релизах не пишется, нужна для внутреннего использования)
Тип релиза - может принимать одно из следующих значений:
	Pre-alpha - внутренний релиз, выпущенный до этапа альфа тестирования
	Alpha - релиз на внутреннее тестирование, проверка работоспособности внутренними тестировщиками.
	Beta - релиз на публичное тестирование, отправка на тестирование внешним тестировщикам, потенциальным пользователям.
	RC (Release candidate) - стабильная версия готовая стать полноценным релизом, оставленная для последней комплексной проверки мелких упущенных ошибок.
	RTM (Release to manufacturing) - выпуск продукта готового к тиражированию.

Примеры:
Version: 1.1 - Очередной релиз.
Version: 1.2.1 - Выпуск первого пакета багфиксов к версии 1.2
Version: 1.3.0.19fcc60122 - внутренний выпуск.
Version: 1.4.0 Beta - релиз для бета тестирования.
Version: 1.4.4 RTM - релиз в продакшн.
